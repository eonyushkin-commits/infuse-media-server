# 🎬 TorrServer to Infuse Bridge

Автоматизированный `.strm` + WebDAV мост между **TorrServer** и плеером **Infuse** (Apple TV, iOS, Mac) для потокового воспроизведения торрентов без предварительного скачивания файлов на диск.

## ⚙️ Архитектура и принцип работы

Проект поднимает несколько сервисов в Docker:

- **TorrServer** — источник торрент-раздач и HTTP-стриминга.
- **TorrServer proxy (Nginx)** — HTTP‑прокси перед TorrServer с Basic Auth, через который Infuse обращается к потокам.
- **Parser** — сервис, который опрашивает TorrServer и генерирует `.strm`-файлы.
- **WebDAV** — публикует каталог `.strm`-файлов для подключения в Infuse.

Как это работает:

1. Вы добавляете раздачу в TorrServer.
2. Parser обнаруживает её при очередном опросе.
3. Parser создаёт `.strm`-файлы в библиотеке проекта.
4. WebDAV публикует эту библиотеку для Infuse.
5. Infuse индексирует файлы и воспроизводит поток через TorrServer proxy.

## ✨ Возможности

- Полностью контейнерный запуск через Docker.
- Установка и обновление через один `install.sh` (без ручной работы с Git и Compose).
- Автоматическая генерация `.strm`-файлов для Infuse с атомарной записью (без риска пустых файлов при чтении Infuse).
- Фоновый опрос TorrServer без cron и Python-зависимостей на хосте.
- Публичная библиотека через WebDAV с HTTP Basic Auth.
- Хранение конфигурации и данных внутри каталога проекта (`.env`, база TorrServer, библиотека `.strm`).
- Возможность безопасного обновления кода без потери данных в `./ts` и `./strm_library`.

## 📋 Требования

На сервере должны быть установлены:

- `git`
- `docker`
- `curl`
- Docker Compose v2 или совместимый `docker-compose`

Скрипт установки должен запускаться от `root` или через `sudo`.

## 🚀 Установка

Запустите на сервере:

```bash
curl -O https://raw.githubusercontent.com/eonyushkin-commits/torrserver-infuse-bridge/refs/heads/main/install.sh
chmod +x install.sh
./install.sh
```

Скрипт установки:

- клонирует репозиторий в `/opt/`;
- запрашивает параметры WebDAV и TorrServer;
- создаёт `.env` с указанными значениями;
- генерирует `.htpasswd` и конфиг `nginx.conf` для прокси перед TorrServer;
- запускает контейнеры через Docker Compose.

## 🔄 Обновление

Повторный запуск `install.sh` является штатным способом обновления проекта и изменения параметров:

```bash
cd /opt/
./install.sh
```

Если каталог `/opt/` является git‑репозиторием, скрипт:

- остановит текущие контейнеры;
- определит текущую ветку (`git rev-parse --abbrev-ref HEAD`);
- выполнит `git fetch --all && git reset --hard "origin/$ТЕКУЩАЯ_ВЕТКА"` для обновления кода;
- при необходимости пересоздаст `.env` и конфиги;
- поднимет контейнеры с актуальной конфигурацией, **не трогая каталоги** `./ts` и `./strm_library`.

## 🧩 Параметры установки

Во время установки скрипт интерактивно запрашивает:

- `WEBDAV_PORT` — внешний порт WebDAV, по умолчанию `8080`;
- `WEBDAV_USER` — логин WebDAV и TorrServer proxy, по умолчанию `admin`;
- `WEBDAV_PASSWORD` — пароль WebDAV и TorrServer proxy;
- `HOST_IP` — внешний IP-адрес сервера (или доменное имя).  
  **Важно:** именно этот адрес записывается внутрь генерируемых `.strm`-файлов.  
  Если указать его неверно (например, оставить `127.0.0.1` при установке на удаленный VPS), Infuse успешно загрузит медиатеку по WebDAV, но не сможет запустить воспроизведение видео;
- `TORR_PORT` — внешний порт TorrServer (порт прокси‑контейнера). По умолчанию скрипт предлагает случайный свободный порт в диапазоне `10000–60000`, но вы можете указать любой другой незанятый порт.

Эти значения сохраняются в файл `.env`, который используется при запуске сервисов.

## 📁 Структура проекта

После установки проект работает из каталога:

```bash
/opt/torrserver-infuse-bridge
```

Основные рабочие каталоги и файлы:

- `install.sh` — установка, обновление и переконфигурация (идемпотентный скрипт развёртывания);
- `.env` — параметры окружения (порт WebDAV/TorrServer, логин/пароль, внешний IP/домен);
- `.htpasswd` — файл с bcrypt‑хэшем пароля для HTTP Basic Auth (генерируется автоматически, не хранится в Git);
- `nginx.conf` — конфигурация Nginx‑прокси перед TorrServer (генерируется автоматически);
- `./torr_to_strm.py` — скрипт парсера, который опрашивает TorrServer и создаёт `.strm`‑файлы;
- `./strm_library` — библиотека `.strm`-файлов для Infuse;
- `./ts` — данные TorrServer (конфигурация, база, кэш);
- `docker-compose.yml` — описание контейнеров проекта.

## 🍏 Подключение в Infuse

1. Откройте Infuse → **Настройки (Settings)** → **Добавить файлы (Add Files)** → **Другое... (Other...)**.
2. Выберите протокол **WebDAV**.
3. Заполните данные, используя значения, которые вы ввели при запуске `install.sh`:
   - **Адрес (Address):** IP вашего VPS или доменное имя (без `http://`).
   - **Пользователь (Username):** ваш логин (указанный при установке, по умолчанию `admin`).
   - **Пароль (Password):** ваш пароль (указанный при скрытом вводе).
4. В разделе **Дополнительно (Advanced)** укажите:
   - **Порт (Port):** порт WebDAV (указанный при установке, по умолчанию `8080`).
5. Нажмите **Сохранить (Save)** и добавьте папку в Избранное (⭐).

Infuse автоматически просканирует файлы, подтянет постеры, описания и сгруппирует сериалы по сезонам. Новые фильмы будут появляться в Infuse с небольшой задержкой после добавления в TorrServer и обработки парсером.

## 🛠 Администрирование

Проверить состояние контейнеров:

```bash
cd /opt/torrserver-infuse-bridge
docker compose ps
```

Посмотреть логи parser:

```bash
docker logs -f strm-parser
```

Остановить сервисы:

```bash
cd /opt/torrserver-infuse-bridge
docker compose down
```

Запустить сервисы снова:

```bash
cd /opt/torrserver-infuse-bridge
docker compose up -d
```

## 🌐 Полезные URL

После установки обычно используются два основных адреса:

- TorrServer UI (через Nginx proxy): `http://HOST_IP:TORR_PORT`
- WebDAV URL: `http://HOST_IP:WEBDAV_PORT`

Оба сервиса защищены HTTP Basic Auth с логином/паролем, заданными при установке.

## 🔐 Безопасность

- Файл `.env` создаётся с правами `600` и не коммитится в репозиторий.
- Файл `.htpasswd` содержит bcrypt‑хеш пароля, генерируется контейнером `httpd:alpine` и также не хранится в Git.
- Доступ к WebDAV и TorrServer proxy защищён логином и паролем, которые задаются во время установки.
- TorrServer не публикуется напрямую наружу: внешние подключения идут через Nginx‑прокси с Basic Auth.
- `.strm`‑файлы содержат URL вида `http://user:password@host:port/...`, но доступны только через WebDAV, который сам требует те же самые креденшелы.
- Рекомендуется использовать сложный пароль и при необходимости ограничивать внешний доступ к портам брандмауэром/Firewall (ufw, Security Groups и т.п.).

## 📝 Примечания

- Проект ориентирован на полностью контейнерный запуск без ручной установки Python-окружения на хосте.
- Библиотека для Infuse публикуется из локального каталога `./strm_library`, данные TorrServer — из `./ts`.
- Корректность отображения медиатеки в Infuse зависит от того, как parser формирует `.strm`-файлы и имена элементов; названия файлов приводятся к удобному для Infuse виду.
- Parser использует устойчивую к ошибкам логику опроса и атомарную запись `.strm` через временный файл, чтобы Infuse не сталкивался с пустыми или частично записанными файлами.
- Проект особенно удобен для VPS-сценария, где TorrServer, генератор `.strm`, Nginx‑прокси и WebDAV работают как единый стек.
