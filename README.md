# 🎬 TorrServer to Infuse Media Server

Автоматизированный, полностью контейнеризированный медиасервер для потокового воспроизведения торрентов (фильмов и сериалов) в плеере **Infuse** (Apple TV, iOS, Mac) без предварительного скачивания файлов на диск.

## ⚙️ Архитектура и принцип работы

Проект поднимает три сервиса в Docker:

- **TorrServer** — источник торрент-раздач и HTTP-стриминга.
- **Parser** — сервис, который опрашивает TorrServer и генерирует `.strm`-файлы.
- **WebDAV** — публикует каталог `.strm`-файлов для подключения в Infuse.

Как это работает:

1. Вы добавляете раздачу в TorrServer.
2. Parser обнаруживает её при очередном опросе.
3. Parser создаёт `.strm`-файлы в библиотеке проекта.
4. WebDAV публикует эту библиотеку для Infuse.
5. Infuse индексирует файлы и воспроизводит поток через TorrServer.

## ✨ Возможности

- Полностью контейнерный запуск через Docker.
- Установка и обновление через один `install.sh`.
- Автоматическая генерация `.strm`-файлов для Infuse.
- Фоновый опрос TorrServer без cron и Python-зависимостей на хосте.
- Публикация библиотеки через WebDAV.
- Хранение конфигурации и данных внутри каталога проекта.

## 📋 Требования

На сервере должны быть установлены:

- `git`
- `docker`
- `curl`
- Docker Compose v2 или совместимый `docker-compose`

Скрипт установки должен запускаться от `root` или через `sudo`.

## 🚀 Установка

Запустите на сервере:

```bash
curl -O https://raw.githubusercontent.com/eonyushkin-commits/infuse-media-server/refs/heads/main/install.sh
chmod +x install.sh
./install.sh
```

Скрипт установки:

- клонирует репозиторий в `/opt/infuse-media-server`;
- запрашивает параметры WebDAV и TorrServer;
- создаёт `.env`;
- запускает контейнеры через Docker Compose.

## 🧩 Параметры установки

Во время установки скрипт интерактивно запрашивает:

- `WEBDAV_PORT` — внешний порт WebDAV, по умолчанию `8080`;
- `WEBDAV_USER` — логин WebDAV, по умолчанию `admin`;
- `WEBDAV_PASSWORD` — пароль WebDAV;
- `HOST_IP` — внешний IP-адрес сервера (или доменное имя). **Важно:** именно этот адрес записывается внутрь генерируемых `.strm`-файлов. Если указать его неверно (например, оставить `127.0.0.1` при установке на удаленный VPS), Infuse успешно загрузит медиатеку по WebDAV, но не сможет запустить воспроизведение видео;
- `TORR_PORT` — внешний порт TorrServer, по умолчанию `8090`.

Эти значения сохраняются в файл `.env`, который используется при запуске сервисов.

## 📁 Структура проекта

После установки проект работает из каталога:

```bash
/opt/infuse-media-server
```

Основные рабочие каталоги и файлы:

- `install.sh` — установка, обновление и переконфигурация;
- `.env` — параметры окружения;
- `./torr_to_strm.py` — скрипт парсера;
- `./strm-library` — библиотека `.strm`-файлов для Infuse;
- `./ts` — данные TorrServer;
- `docker-compose.yml` — описание контейнеров проекта.

## 🔄 Обновление

Повторный запуск `install.sh` является штатным способом обновления проекта и изменения параметров:

```bash
cd /opt/infuse-media-server
./install.sh
```

Если репозиторий уже существует, скрипт обновит код и пересоздаст контейнеры с актуальной конфигурацией.

## 🍏 Подключение в Infuse

1. Откройте Infuse → **Настройки (Settings)** → **Добавить файлы (Add Files)** → **Другое... (Other...)**.
2. Выберите протокол **WebDAV**.
3. Заполните данные, используя значения, которые вы ввели при запуске `install.sh`:
   - **Адрес (Address):** IP вашего VPS (без `http://`).
   - **Пользователь (Username):** ваш логин (указанный при установке, по умолчанию `admin`).
   - **Пароль (Password):** ваш пароль (указанный при скрытом вводе).
4. В разделе **Дополнительно (Advanced)** укажите:
   - **Порт (Port):** порт WebDAV (указанный при установке, по умолчанию `8080`).
5. Нажмите **Сохранить (Save)** и добавьте новую папку в Избранное (⭐).

Infuse автоматически просканирует файлы, подтянет постеры, описания и сгруппирует сериалы по сезонам. Новые фильмы будут появляться в Infuse с небольшой задержкой после добавления в TorrServer.

## 🛠 Администрирование

Проверить состояние контейнеров:

```bash
cd /opt/infuse-media-server
docker compose ps
```

Посмотреть логи parser:

```bash
docker logs -f strm-parser
```

Остановить сервисы:

```bash
cd /opt/infuse-media-server
docker compose down
```

Запустить сервисы снова:

```bash
cd /opt/infuse-media-server
docker compose up -d
```

## 🌐 Полезные URL

После установки обычно используются два основных адреса:

- TorrServer UI: `http://HOST_IP:TORR_PORT`
- WebDAV URL: `http://HOST_IP:WEBDAV_PORT`

## 🔐 Безопасность

- Файл `.env` создаётся с ограниченными правами доступа.
- Доступ к WebDAV защищён логином и паролем, которые задаются во время установки.
- Поскольку WebDAV и TorrServer публикуются наружу через порты хоста, рекомендуется использовать сильный пароль и ограничивать внешний доступ сетевыми правилами.

## 📝 Примечания

- Проект ориентирован на полностью контейнерный запуск без ручной установки Python-окружения на хосте.
- Библиотека для Infuse публикуется из локального каталога `./strm-library`.
- Корректность отображения медиатеки в Infuse зависит от того, как parser формирует `.strm`-файлы и имена элементов.
- Проект особенно удобен для VPS-сценария, где TorrServer, генератор `.strm` и WebDAV работают как единый стек.
